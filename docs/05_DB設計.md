##　5‐3. ER図
本システムにおけるテーブル間の関係は以下の通りである。
### エージェントと面接
エージェントは複数の面接を担当する
1つの面接には1つのエージェントが紐づく
エージェント⇒面接(1 : N)

### 企業と面接
企業は複数回面接を担当する
1つの面接には、1つの企業が紐づく
企業⇒面接(1 : N)

### 求職者と面接
求職者は複数回面接を実施する可能性がある
1つの面接には、1人の求職者が紐づく
求職者⇒面接(1 : N)

### 面接と候補日
1つの面接に対して、複数の候補日が登録される
候補日は必ず1つの面接が紐づく
面接⇒候補日(1 : N)

### 候補日と回答
1つの候補日に対して、企業及び求職者の参加可否が登録される
回答は必ず1つの候補日が紐づく
候補日⇒回答(1 : N)
