# ER図とは・・？
テーブル同士のつながり地図
3つの要素；
| No | 要素 | 説明 |記載法|
|----|------|------|------|
|1.|エンティティ（Entity）|テーブルのこと(企業や求職者)|四角で表すことが多い|
|2.|リレーション（Relationship）|テーブル同士の関係|⇒で表すことが多い|
|3.|カーディナリティ（数の関係）|テーブル同士の“人数関係|1 : 1,1 : N,N : M|

# ER図の正規化チェック
テーブル攻勢に矛盾や無駄がないかチェック
SQLを使う、VSコードにて実施

VARCHAR(100や250)などの数字を入れる理由
①無限に入れないため
②DBを軽く・速くする(上限を決めておくとDBがメモリを管理しやすくなる)
よく使われる数＋型セット⇒
| 型 | 意味 | 数字|
|VARCHAR(50)|短い文字|50文字|
|VARCHAR(100)|名前など|100文字|
|VARCHAR(225)|メールアドレスやURL等|225文字|

NOT NULL＝空欄NG
